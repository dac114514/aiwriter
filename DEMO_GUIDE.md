# AI Story Weaver - 项目演示指南

## 🎯 演示目标

展示AI Story Weaver项目的完整功能流程，从故事配置到阅读体验的全过程。

## 📱 演示流程

### 第一阶段：应用启动 (30秒)

**演示内容**:
1. 应用启动画面显示"AI故事创作"
2. 主界面展示原有AI Chat功能入口
3. 新增"AI故事创作"按钮在适当位置

**技术亮点**:
- Material Design 3主题
- 流畅的启动动画
- 响应式布局适配

### 第二阶段：Setup阶段演示 (2分钟)

#### 步骤1: 进入故事配置 (15秒)
```
用户操作: 点击"AI故事创作"按钮
界面变化: 进入StorySetupScreen
展示内容: "选择故事类型"标题和6个模板卡片
```

**演示要点**:
- 6种故事类型清晰展示
- 每个模板包含名称、描述、难度标签
- 已选模板高亮显示

#### 步骤2: 选择模板 (15秒)
```
用户操作: 点击"奇幻冒险之旅"模板
界面变化: 模板选中状态更新
展示内容: 选中状态的视觉反馈
```

**演示要点**:
- 选中状态的视觉区分
- 模板信息完整展示
- 交互反馈及时响应

#### 步骤3: 自定义设定 (45秒)
```
用户操作: 填写角色设定、背景描述、特殊要求
界面变化: 输入框获得焦点，实时验证
展示内容: 分步骤引导界面
```

**演示要点**:
- 清晰的步骤指示器
- 合理的输入提示
- 友好的错误提示（如果有）

#### 步骤4: 生成大纲 (30秒)
```
用户操作: 点击"生成大纲"按钮
界面变化: 显示加载动画，调用AI API
展示内容: "正在生成故事大纲..."进度提示
```

**演示要点**:
- 真实的API调用过程
- 加载状态的友好提示
- 网络请求的成功/失败处理

#### 步骤5: 大纲预览 (30秒)
```
用户操作: 查看生成的故事大纲
界面变化: 大纲内容详细展示
展示内容: 故事标题、概要、角色、情节结构
```

**演示要点**:
- 大纲内容的完整性
- 信息的层次化展示
- 用户的确认操作

#### 步骤6: 开始创作 (15秒)
```
用户操作: 点击"开始创作"按钮
界面变化: 导航到StoryScreen
展示内容: "准备开始创作"确认界面
```

**演示要点**:
- 创建过程的完整性
- 用户确认的重要性
- 平滑的页面转场

### 第三阶段：故事阅读演示 (3分钟)

#### 步骤1: 章节阅读 (60秒)
```
用户操作: 按章节顺序阅读故事内容
界面变化: 章节内容逐步显示
展示内容: 第1章故事正文，Markdown渲染
```

**演示要点**:
- 章节标题和内容的清晰展示
- Markdown文本的正确渲染
- 舒适的阅读体验

#### 步骤2: 进度追踪 (30秒)
```
用户操作: 观察顶部进度条
界面变化: 实时更新阅读进度
展示内容: 第X章/共Y章，完成百分比
```

**演示要点**:
- 进度指示器的准确性
- 实时更新的流畅性
- 视觉反馈的及时性

#### 步骤3: 分支点识别 (30秒)
```
用户操作: 滚动到第8章附近
界面变化: 检测到分支点，显示选择标签
展示内容: "分支点"标签和选择提示
```

**演示要点**:
- 智能的分支点检测
- 明显的视觉标识
- 用户预期的满足

#### 步骤4: 做出选择 (45秒)
```
用户操作: 点击其中一个选择项
界面变化: 选择项高亮，箭头图标显示
展示内容: 3-4个合理的选择选项
```

**演示要点**:
- 选择项的视觉设计
- 交互的直观性
- 选择的明确反馈

#### 步骤5: 继续故事 (30秒)
```
用户操作: 点击"继续"按钮
界面变化: 生成下一章内容
展示内容: 基于选择的新章节内容
```

**演示要点**:
- 选择影响故事走向
- 新内容的连贯性
- 加载状态的友好提示

#### 步骤6: 重新开始 (15秒)
```
用户操作: 点击"重新开始"按钮
界面变化: 清除当前进度，返回设置
展示内容: 进度重置，准备重新体验
```

**演示要点**:
- 重新开始功能的可用性
- 进度清除的确认机制
- 用户体验的一致性

### 第四阶段：技术架构展示 (2分钟)

#### 数据流演示 (60秒)
```
展示内容: 数据从UI到底层的技术流动
技术要点:
1. UI事件触发ViewModel方法
2. ViewModel调用Repository获取数据
3. Repository调用StoryGenerator生成内容
4. AI API返回结果并更新UI状态
```

**演示要点**:
- MVVM架构的清晰展示
- 数据流的单向性
- 各层的职责分离

#### 状态管理演示 (60秒)
```
展示内容: StateFlow和响应式UI的工作原理
技术要点:
1. 用户操作改变StateFlow值
2. ViewModel收集StateFlow变化
3. Compose自动重新渲染UI
4. 状态同步的实时性
```

**演示要点**:
- 响应式编程的优势
- 状态管理的简洁性
- UI与数据的自动同步
